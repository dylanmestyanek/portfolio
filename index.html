Macro Calculator

Description: A web application that estimates the user's ideal daily calorie and macronutrient intake to support their nutrition goals.

Challenges: The most challenging aspect of this project was incorporating data visualization with different forms of graphs, and implementing user authentication using JWTs. It proved to be challenging, yet an extremely fun learning experience.

Lessons learned: This project taught me organization and structure for React applications. As the application spread out, I learned the importance of planning Redux implementation to ensure clean state management.

Tech Stack: React | Redux | Axios

Role: Front End Engineer

JavaScript Voice Timer

Description: While maintaining the standard functionality of a timer application, this vanilla JavaScript timer implements the Google Chrome's voice recognition functionality to interact with the timer.

Challenges: The majority of the time was spent dissecting the response object from the Speech Recognition API, and discovering ways to manipulate the text into commands for the application to respond to.

Lessons learned: This project was one of the first useful JavaScript applications I worked on. The construction of this project tested my creativity and provided me with my first taste of the JavaScript foundation, such as DOM manipulation and Array/Object methods. The experience of building this app taught me patience, persistence, and planning.

Tech Stack: HTML | CSS | JavaScript

Role: Front End Engineer

Spotifier Song Suggester

Description: A web application that allows users to search their favorite songs, build playlists, and discover songs they've never heard before.

Challenges: Learning to implement a convenient user flow throughout the application using React Router and collaborate with team members differing opinions on the application's user interface.

Lessons learned: This project taught me effective team communication and exposed me to React's power and efficiency. Working with a pair of Data Scientists taught me to speak their language, follow their thought process, and understand their data structure being provided to the back-end engineers.

Role: Front End Engineer

Weight Lifting Journal API

Description: RESTful API that allows users to create workouts tailored to their lifestyle.

Challenges: Planning an efficient SQL database schema, and constructing endpoints to return the appropriate data structure. Testing the server, using Jest, to ensure endpoints were providing expected results.

Lessons learned: Gained hands-on experience with designing a database schema and discovered the importance of clear, concise endpoint documentation. Learned to manipulate returned data to suit the front-end developers needs, and assisted debugging the React application.

Tech Stack: Node | Express | Knex | PostgreSQL | JSON Web Token

Role: Back End Engineer

Sauti Databank

Description: A data visualization project for Sauti.org. Sauti manages an SMS platform that East African border traders use to look up realtime market data. We converted that data, collected from SMS queries and produced it into a readable and then visual format.

Challenges: Refactoring the Node server from a RESTful API into GraphQL. Wiping the entire React application down to bare bones, and rebuilding an increasingly more efficient, and dynamic front-end application.

Lessons learned: Planning each step of a project provides a great sense of direction for the entire team. Pre-made plans allows for efficicent task delegation. Taking a step back, cleansing inefficient code, proves to make progress in the future a simpler feat.

Tech Stack: React | GraphQL | Node | Express | MySQL | Nivo

Role: Front End Engineer
